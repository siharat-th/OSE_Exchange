inlinestatic

Encodes the given data into a 64-bit value. This is a super important concept- we encode some of our callback and hedge data into a 64-bit value and pass it as part of the order to the exchange. This is a way to pass data to the exchange, have the exchange persist it, and then have it sent back to us in the execution report. We can encode various data into this 64-bit value and then decode it when we get it back in the execution report. We like this because we use a 64-bit integer rather than a string, which is orders of magnitude slower to decode/decipher and make quick decisions on. So, in CME, we use the OrderRequestId field to pass this data back and forth, which is a 64-bit integer that we assign and it "lives" with the order throughout its order cycle, enabling us to efficiently pass data back and forth.
